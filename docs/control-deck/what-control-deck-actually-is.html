Briefing 01 · Product architecture

Control Deck is a full operating environment, not a single-purpose harness.

The right mental model is not “chat app,” not “trainer,” and not merely “eval harness.” Control Deck is a local-first human+agent operating surface that combines UI shell, agent runtime, inference router, machine-control layer, creative workstation, and persistence spine into one product.

Observed in repo: 19 deck pages 65 pane components 73 API routes 226 TypeScript files under lib/

The thesis

Control Deck is best understood as the operating layer for an entire machine and AI stack. Training and evaluation matter, but they should be built around this environment rather than replacing its identity.

Repo-grounded evidence: README calls it “one cockpit for every surface your agent needs to touch.” DESIGN.md says “Control Deck is not a chatbot. It is the operating layer for an entire machine and AI stack.” SURFACE.md says the primary content is work product and agent activity, not text exchange.

What it is, in layers

1. Operator cockpit

The app has dedicated deck surfaces for chat, models, hardware, tools, runs, settings, voice, visual work, live music, terminal, workspace, and more. That makes it a supervision and control shell for both humans and agents, not just a single chat transcript.

2. Agent runtime surface

The run lifecycle spans chat submission, tool dispatch, approval, interruption, resume, artifact rendering, and persistence. In other words: the UI is not merely showing results; it is presenting an active runtime.

3. Inference and model control plane

The repo has a large inference/hardware surface: provider bindings, metrics, suggestions, system profiling, benchmarks, embeddings, rerank, image/audio/video/vision/tts/stt routes. That is model orchestration, not just prompt dispatch.

4. Machine-control layer

Tools reach the terminal, files, browser-like surfaces, native OS accessibility APIs, code execution sandboxes, and workspace flows. This makes Control Deck a machine interface, not only a model interface.

5. Multimodal creative workstation

Voice, audio, ComfyUI, visual, live transport, mixer, playlist, pattern rack, widgets, and artifact views make the product broader than conventional agent consoles. It is built for making things, not just asking questions.

6. Desktop application platform

Electron, native dependencies, themed browser windows, and OS adapters make the app a real desktop environment with system reach, not just a local website.

What it is not

Reduction Why it is incomplete Better framing
“It’s a chatbot UI.” The docs explicitly push against this. Agent activity, tools, artifacts, runs, approvals, and work product outrank plain conversation. It is a work-oriented control surface with chat as one lane.
“It’s an eval harness.” Harness behavior is important, but the product also includes native control, hardware visibility, multimodal generation, desktop embedding, and operator workflows. It is a product with an eval substrate inside it.
“It’s a trainer.” Nothing in the current repo suggests that gradient updates, checkpoint management, or distributed training infrastructure are core responsibilities. It should feed external training stacks by generating traces, labels, and tasks.
“It’s only a tools bridge.” The repo has much more: panes, routes, model routing, desktop wiring, media surfaces, skills, plugins, rules, runs, and artifact storage. It is a unified operating environment where the bridge is one backbone.

Why this framing matters

Product

If you flatten Control Deck into a narrow harness, you lose the thing that makes it worth training against: a rich, cross-surface environment with actual operator workflows.

Research

If you preserve its full shape, you can study more realistic agent behavior: model switching, approval-seeking, artifact production, hardware-aware decisions, and multi-tool coordination.

Training

The goal is not to teach a policy to hit one endpoint. The goal is to teach a policy to operate a layered environment safely and effectively.

The best one-sentence description

Control Deck is a local-first operating environment for human+agent work: a UI shell, runtime, model router, machine-control plane, artifact system, and multimodal workstation that can later serve as a high-value environment for training and evaluating models.

That preserves the product’s real identity while still making room for the training/eval agenda.
