What it stands for

Six pillars, one cockpit.

Not a feature list — a stance. Every screen, event, and shortcut in Control Deck serves one of these. If it doesn't, it isn't in the deck.

01

Aggregate

Ollama, ComfyUI, SearxNG, Qdrant, the voice sidecar, Agent-GO — six local services become one cockpit, one bearer token, one SQLite file you can back up with cp. This isn't an app launcher. It's a single composed surface where the services know about each other.

02

Control

Reads your VRAM. Watches your services. Drives native apps via AT-SPI / AX / UIA. Attaches to Chromium via CDP. The deck sees your hardware and can move your cursor — not just your API tokens.

03

Connect

Every modality shares one timeline, one artifact store, one run history. Voice asks; vision answers; code executes; 3D renders — in the same conversation, stored in the same database, addressable by the same agent.

04

Synergize

Ten surfaces, zero context loss. The dashboard knows what the chat is doing. The browser knows what the agent clicked. The voice engine knows which model is loaded. Meta-state is ambient, not a setting you go find.

05

Multi-modal

Text, voice, vision, image-gen, 3D, audio, video, code, web, native OS — all first-class citizens. Not "supports X via plugin" — rendered in the same timeline, stored as the same artifact kind, invoked by the same agent.

06

Ease

One AppImage. One Cmd+K. One bearer token. One SQLite you back up with cp. The cockpit is harder to lose than to set up.

Synergy · the compound unlock

The cockpit isn't the sum of its surfaces. It's the product.

Aggregation is table stakes. Synergy is the payoff. When every modality shares one timeline, every service reads one SQLite, and every surface listens to one event bus, the number of flows you can run compounds — not adds. Ten capabilities bolted onto one cockpit yields far more than ten capabilities in ten tabs.

Flow 01 · Axis A

Research → Summary → Cite

  1. 🎙 Voice — push-to-talk: "find the latest on WebGPU compute shaders."
  2. 🔍 SearxNG — agent searches your local metasearch, not a cloud API.
  3. 🌐 Themed Chromium — top 3 results open as in-deck browser windows.
  4. 📝 Canvas — Monaco opens with a generated markdown digest, cited.
  5. 🧠 Qdrant — digest embedded into your vector DB for next time.

Nothing left the deck. Nothing left your machine.

Flow 02 · Axis A

Prompt → Image → 3D → Voice

  1. ✍ Chat — "sketch a lantern, make it rotate, narrate the design."
  2. 🎨 ComfyUI — your workflow runs on local VRAM.
  3. 🎲 image-to-3D — a built-in tool lifts it to a GLB.
  4. 🔊 Voice sidecar — piper / xtts / chatterbox narrates.
  5. 🧩 One artifact card — image + 3D + audio, all siblings.

Four modalities, one conversation turn, one place to find it again.

Flow 03 · Axis B

Screenshot → Understand → Act

  1. 🖼 Native — AX adapter screenshots whatever native app you're in.
  2. 👁 Vision slot — your chosen vision model extracts structured state.
  3. 🧭 Agent-GO — plans a three-step correction based on the state.
  4. 🖱 AT-SPI / AX / CDP — clicks, types, and navigates to apply it.
  5. ✅ Timeline — every click is a replayable ToolCallResult.

Computer-use without a cloud round-trip, with an audit trail.

Flow 04 · Axis B

Observe → Adapt

  1. 📈 VRAM widget — flags GPU 0 at 98%, sustained.
  2. 🧭 Agent-GO — proactively suggests: swap FAST slot to qwen2.5-7b.
  3. ⚙ Provider runtime — setRuntimeProvider() hot-swaps, no restart.
  4. 📟 StatusStrip — shows "FAST: llama3.1 → qwen2.5:7b".
  5. ▶ Run continues — mid-turn, no data loss, no fresh thread.

The cockpit reads its own gauges and acts on them.

Flow 05 · Axis C

Recall → Synthesize

  1. ⌘ Cmd+K — "what have I made about WebGPU?"
  2. 🔎 Qdrant — vector search across chats, artifacts, run events.
  3. 📚 Agent — receives top-k bundles, each tagged with source runId.
  4. 🖋 Canvas — synthesizes a brief; every citation is a live link.
  5. ⏱ Click a citation — the original timeline reopens at that exact turn.

Long-term memory that shows its work.

Flow 06 · Axis C

Clip → Reason → Replay

  1. 🖱 Themed Chromium — right-click on any page → "Send to thread."
  2. 📎 UploadTray — URL + extracted text arrive as an attachment.
  3. 🧠 Agent — reasons over it, cites paragraphs, flags what it doubts.
  4. 💾 SQLite — the whole exchange is one run, one row.
  5. 🔁 /deck/runs — next week, replay it with a different model.

Anything you see becomes context. Any context becomes a replay.

Why synergies are cheap here

In the deck, N × N interactions cost N + N to build.

Most stacks wire each integration by hand — chat ↔ voice, voice ↔ search, search ↔ vector, vector ↔ canvas. That's quadratic work. Control Deck makes every modality an event on one AG-UI bus, every artifact a row in one table, every surface a consumer of one React context. Add a new modality and it inherits every existing synergy on the same day. This is why the flows above compose — and why the next one you imagine probably already works.

  • One event protocol — AG-UI over SSE
  • One artifact store — UUID-named, mime-tagged, DB-indexed
  • One timeline — discriminated-union segments render anything
  • One provider abstraction — the slot knows which model answers
  • One run history — every synergy is a replayable trace
  • One shortcut — Cmd+K finds any of the above

OS & browser reach

  • Linux AT-SPI accessibility tree via atspi-helper.py; portal key/type/click_pixel via remote-desktop.py daemon.
  • macOS AXUIElement + CGEvent via compiled macos-ax-helper.bin.

  • Chromium Every themed browser window is a first-class CDP target. browser-harness attaches over the opt-in DevTools port.
  • Terminal PTY-backed terminal surface via standalone terminal-service.

The whitespace

No current peer combines service aggregation (Ollama · ComfyUI · SearxNG · Qdrant · voice) with genuine multi-platform OS widget control (AT-SPI / AX / UIA) and Chromium automation (CDP) in a single local artifact. Open WebUI is the dominant homelab aggregator but lives entirely in a browser tab. AnythingLLM gestures at OS reach but its computer-use is screenshot-based via a cloud API — no structured accessibility tree, no offline mode. NVIDIA NemoClaw is the only other project aiming at "always-on local agent" — GPU-vendor-locked, server-focused, no GUI.

Control Deck occupies a category without an incumbent: a local-first, hardware-agnostic cockpit where the AI can talk to your screen, your browser, your image generator, and your voice pipeline simultaneously.

Start

Run the cockpit in five commands.

# clone and install
git clone <control-deck> && cd control-deck
bun install

# configure — provider keys, DECK_TOKEN, Ollama URL
cp .env.example .env.local

# bring up the full local stack (Agent-GO, Ollama, ComfyUI, Searx, voice)
./start-full-stack.sh

# dev
bun run dev        # http://localhost:3333

# or package as an AppImage
bun run electron:pack
